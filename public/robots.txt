# ============================================================================
# CONFIGURATION DOCUMENTATION
# ============================================================================

# PRIORITY LOGIC:
# 1. Most specific rules apply first
# 2. Allow rules can override Disallow rules if placed first
# 3. Empty value means allow/disallow everything
# 
# EXAMPLES:
# Disallow: /admin/        # Blocks /admin/anything
# Disallow: /admin         # Blocks /admin but NOT /admin/ or /admin-something
# Disallow: /*.pdf         # Can use * as wildcard
# Disallow: /*?id=*        # Blocks pages with query parameter
# 
# COMMON PATTERNS:
# Disallow: /              # Blocks everything (use with caution!)
# Disallow:                # Allows everything (empty rule)
# Allow: /public           # Whitelist a specific path
# 
# BEST PRACTICES:
# 1. Be explicit - block specific paths rather than broad patterns
# 2. Don't block CSS, JS, images needed to render pages
# 3. Use Allow rules to override Disallow when needed
# 4. Test changes with Google Search Console robots.txt Tester
# 5. Monitor crawl stats in Google Search Console
#

# ============================================================================
# SECURITY NOTES
# ============================================================================

# IMPORTANT: robots.txt is NOT a security measure!
# - It's a guideline that well-behaved bots follow
# - Malicious actors can ignore it
# - Never put sensitive data on pages you block in robots.txt
# - Use .htaccess, authentication, or noindex meta tags for security
#
# For sensitive areas:
# 1. Use robots.txt (courtesy request)
# 2. Add authentication (actual security)
# 3. Use robots: noindex meta tag (prevents indexing)
# 4. Use X-Robots-Tag header (prevents indexing)
